## Problem 1-Pager: MCP 서버 관리 기능 추가

## Background

jindo는 Claude Code의 skills, commands, agents, hooks를 관리하는 CLI 도구입니다. 현재 `jd pkg browse` 명령으로 TUI를 통해 패키지를 탐색하고 원클릭 설치가 가능합니다.

MCP(Model Context Protocol) 서버는 Claude Code의 기능을 확장하는 핵심 요소이지만, 현재 설정이 복잡합니다:

- `~/.claude/settings.json`을 직접 편집해야 함
- 서버 상태(가용 여부, 에러)를 확인하기 어려움
- 인기 있는 서버를 찾고 설치하는 과정이 번거로움

## Problem

MCP 서버 설정과 관리가 수동적이고 복잡합니다:

1. **설정 복잡성**: JSON 파일을 직접 편집해야 하며, 형식 오류 발생 시 Claude Code가 작동하지 않음
2. **상태 파악 어려움**: 서버가 정상 작동하는지, 에러가 있는지 확인하기 어려움
3. **발견 어려움**: 어떤 MCP 서버가 있는지, 어떻게 설치하는지 정보가 분산되어 있음
4. **관리 부재**: 설치된 서버 목록, 로그, 에러 상황을 한눈에 볼 수 없음

## Goal

사용자가 **browse TUI에서 딸깍 한 번으로 MCP 서버를 설치**하고, **서버 상태와 로그를 쉽게 파악**할 수 있게 합니다.

성공 기준:

- [ ] `jd mcp list`: 설치된 MCP 서버 목록과 상태(running/error/stopped) 표시
- [ ] `jd mcp add <서버>`: MCP 서버를 Claude Code 설정에 추가
- [ ] `jd mcp remove <서버>`: MCP 서버를 Claude Code 설정에서 제거
- [ ] `jd mcp status [서버]`: **실시간 TUI 모니터링** - 주기적으로 상태/로그/에러를 자동 갱신하여 표시
  - Bubble Tea 기반 TUI로 구현
  - 주기적 폴링으로 최신 상태 반영 (예: 2초 간격)
  - 서버별 상태 표시: running ✓ / error ✗ / stopped ○
  - 최근 로그 스트리밍 표시
  - 에러 발생 시 상세 메시지 하이라이트
- [ ] `jd mcp browse`: TUI에서 인기 MCP 서버 탐색 및 원클릭 설치
- [ ] `jd guide mcp`: MCP 개념과 활용법 안내

## Non-goals

- **MCP 서버 직접 개발**: jindo는 서버를 관리만 하고, 새 서버 코드를 생성하거나 개발하는 기능은 제공하지 않음
- 원격(SSE/HTTP) MCP 서버 지원은 향후 확장 가능하나 초기 범위 외
- 다른 MCP 클라이언트(Cursor, VSCode 등) 설정 지원은 범위 외

## Constraints

- **기존 TUI 구조 유지**: Bubble Tea 기반의 현재 browse TUI 패턴을 따름 (`internal/tui/browse.go`)
- **Claude Code 설정 파일 직접 수정**: `~/.claude/settings.json`의 `mcpServers` 섹션을 읽고 씀
- **기존 CLI 패턴 준수**: `jd pkg`, `jd skills` 등과 동일한 서브커맨드 구조 사용
